## Assumptions

- 第一版只要求本机开发可用，不做生产打包和静态文件嵌入。
- 第一版允许 duration 显示为 `0:00`，播放可用优先。
- 第一版不做封面、歌词、歌单、实时监听、目录删除和数据库迁移。
- 当前 `config.yaml` 保持：
  - `listen_addr: "127.0.0.1:8080"`
  - `database_path: "./data/music.db"`

## v0.2 页面问题与待做项

当前 v0.1 已经打通扫描、入库、列表展示和播放流，但页面仍然偏“可用壳”，距离日常播放器体验还有一些明显缺口。v0.2 优先解决页面交互、状态反馈和播放器体验问题。

### 播放器体验

- [x] 底部播放器进度条需要接入真实播放进度，而不是固定 `0`。
- [x] 支持拖动进度条跳转播放位置。
- [x] 播放/暂停状态需要完全跟随 audio 事件，处理加载失败、播放结束、用户暂停等情况。
- [x] 上一首、下一首按钮当前没有行为，需要基于当前歌曲列表实现顺序切歌。
- [x] 当前歌曲时长为 `0:00` 时观感较弱，需要在 UI 上区分“未知时长”和真实 `0:00`。
- [x] 点击歌曲后应有加载状态，避免大文件或慢磁盘时用户不知道是否开始播放。

### 歌曲列表

- [x] 歌曲表格缺少空值展示规则，artist/album 为空时应显示更友好的占位，例如 `未知艺术家`、`未知专辑`。
- [x] 当前行选中只表达“当前歌曲”，还需要区分播放中、暂停中、加载中。
- [x] 搜索时没有 loading/error/empty 状态的细分展示。
- [x] 歌曲列表数量增大后需要考虑滚动区域高度和表头固定。
- [ ] 文件名 fallback 的标题中可能包含 `歌名 - 艺术家`，后续可在前端或扫描阶段做轻量拆分。

### 扫描任务面板

- [x] 扫描完成后右侧当前任务区域会消失，只剩最近任务；需要更清晰地保留“最近一次扫描结果”。
- [x] 开始扫描后按钮只显示创建中，缺少“扫描中”禁用状态和防重复提交策略。
- [x] 扫描失败时右侧需要更明显的错误提示，并保留失败目录路径。
- [x] 最近任务列表需要限制高度并滚动，避免任务多时挤压页面。
- [x] 对已扫描目录再次扫描时，应在 UI 上提示这是“重新扫描此目录”，不是新增目录。

### 顶部与导航

- [ ] 左侧导航当前只剩“歌曲”，如果短期没有更多页面，可以考虑进一步压缩侧栏宽度或移除侧栏。
- [x] 顶部媒体库摘要需要在扫描完成后稳定刷新，避免用户误以为扫描没有生效。
- [x] 搜索框需要增加清空按钮。
- [ ] 顶部和右侧区域在小屏宽度下需要响应式折叠规则。

### 状态与错误

- [x] 后端不可用时，前端需要显示明确错误，而不是只有空列表。
- [x] API 请求失败时需要统一错误提示样式。
- [x] 扫描路径为空、目录不存在、无权限时，需要在表单下方显示具体错误。
- [x] 无歌曲、扫描中、扫描完成但无音频文件三种空状态需要文案区分。

### 视觉细节
