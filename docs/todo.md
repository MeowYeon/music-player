## Assumptions

- 第一版只要求本机开发可用，不做生产打包和静态文件嵌入。
- 第一版允许 duration 显示为 `0:00`，播放可用优先。
- 第一版不做封面、歌词、歌单、实时监听、目录删除和数据库迁移。
- 当前 `config.yaml` 保持：
  - `listen_addr: "127.0.0.1:8080"`
  - `database_path: "./data/music.db"`

## v0.2 页面问题与待做项

当前 v0.1 已经打通扫描、入库、列表展示和播放流，但页面仍然偏“可用壳”，距离日常播放器体验还有一些明显缺口。v0.2 优先解决页面交互、状态反馈和播放器体验问题。

### 播放器体验

- [x] 底部播放器进度条需要接入真实播放进度，而不是固定 `0`。
- [x] 支持拖动进度条跳转播放位置。
- [x] 播放/暂停状态需要完全跟随 audio 事件，处理加载失败、播放结束、用户暂停等情况。
- [x] 上一首、下一首按钮当前没有行为，需要基于当前歌曲列表实现顺序切歌。

- [x] 点击歌曲后应有加载状态，避免大文件或慢磁盘时用户不知道是否开始播放。
