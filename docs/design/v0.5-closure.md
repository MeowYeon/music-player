# 聆听 v0.5 收束说明

这份文档用于正式结束 v0.5 阶段，并明确它在整个产品演进中的位置。

## 阶段定位

v0.5 是早期 MVP 路线的终点版本。

它的核心价值不是“完成产品”，而是完成一个可运行、可验证、可持续迭代的本地音乐播放器基础闭环：

- 本地媒体库可添加、可扫描、可重扫。
- 音乐元数据可解析、可索引、可展示。
- 歌曲、歌单、我喜欢、最近播放可以贯通。
- 浏览器前端可以通过底部播放器稳定播放本地音频。
- 基础播放队列、播放模式、排序筛选、多选操作已形成可用工作流。

## v0.5 到此为止

从现在开始，v0.5 不再承担新的产品目标。

它作为一个阶段性版本被保留，用来确认以下事实已经成立：

1. 后端数据模型和扫描闭环已经跑通。
2. 前端和后端的播放器基础协议已经稳定。
3. 本地单用户音乐管理器的最小形态已经被验证。

这意味着后续版本不必再被 MVP 的约束绑住，也不需要继续围绕“够不够最小”来做取舍。

## 不再延续的约束

以下约束只属于 v0.5 及之前的 MVP 阶段，不再默认适用于 v0.6：

- 只做最小页面结构
- 以验证闭环优先于产品表达
- 避免成熟产品级信息架构
- 避免更完整的页面系统
- 避免更强的视觉设计语言
- 避免面向长期使用的产品级交互整理

## 保留资产

虽然 v0.5 结束，但以下资产继续保留，并作为 v0.6 的基础：

- Go 后端、SQLite、本地扫描和流媒体接口
- 当前播放队列和播放模式模型
- 歌曲、歌单、我喜欢、最近播放的核心概念
- 已有 API 语义和本地播放闭环

## 版本切换结论

结论很简单：

v0.5 作为 MVP 阶段正式结束。

从 v0.6 开始，产品目标从“做成一个最小可用播放器”切换为“把聆听做成一个成熟、长期可用、具有完整产品表达的本地音乐产品”。
