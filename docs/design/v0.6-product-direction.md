# 聆听 v0.6 产品方向

## 背景

产品命名从“阿言 / Ayan”切换为“聆听”。

v0.6 不再延续 MVP 心态。它不再只是证明播放器闭环存在，而是进入面向真实长期使用的产品阶段。

## v0.6 的目标

v0.6 的目标不是给 v0.5 补丁式美化，而是重新组织前端产品形态，让聆听开始具备成熟产品应有的结构、秩序和气质。

当前优先级如下：

1. 重构页面信息架构，而不只是调整样式。
2. 重新定义导航、页面职责和全局布局。
3. 统一视觉语言，让界面从“功能拼接”升级为“完整产品”。
4. 为后续更丰富能力预留空间，不再被 MVP 布局绑死。

## 产品阶段变化

v0.5 关注的是：

- 能不能扫
- 能不能播
- 能不能建歌单
- 能不能形成最小闭环

v0.6 开始关注的是：

- 用户每天是否愿意一直打开它
- 信息结构是否清晰、稳定、耐用
- 功能是否各归其位，而不是彼此重复
- 页面是否支持继续扩展，而不是每加一个能力就更乱
- 品牌和视觉是否已经像一个真正产品

## 页面重构原则

### 1. 功能归位

不再用侧边重复堆叠“正在播放 / 扫描状态 / 队列预览”这类信息。

每个能力应该回到最合适的位置：

- 全局播放控制留在底部播放器
- 队列通过播放器入口打开
- 扫描和媒体库管理回到媒体库页面
- 当前页面只展示与当前任务最相关的信息

### 2. 首页不再只是列表容器

歌曲页不应该只是表格外面再包一层壳，而应该成为真正的“主工作区”：

- 展示当前状态
- 承担导航后的第一视图
- 提供明确的浏览、筛选、播放入口
- 保持高频操作的效率

### 3. 导航系统升级

导航不再只是 MVP 式的简单入口集合，而应该成为产品级结构的一部分。

当前已验证的方向是：

- 使用更克制、更强识别度的全局导航
- 减少横向拥挤
- 把内容空间优先留给音乐库本身

### 4. 视觉从“够用”走向“耐用”

v0.6 的界面要避免两种极端：

- 过于朴素，像后台工具
- 过于装饰，影响长期使用

目标是：

- 有明确品牌感
- 信息密度合理
- 适合长时间停留
- 适合桌面场景
- 对未来增加更多页面仍然稳定

## 当前界面方向结论

在本轮讨论里，已经形成一个明确倾向：

- 放弃带右侧重复信息列的布局
- 保留更完整的主工作区
- 优先考虑“无右栏”的成熟布局
- 在“基准无右栏版”和“高级无右栏版”之间，后者更符合 v0.6 的方向

这说明 v0.6 页面重构的起点不是保守修补，而是：

基于“高级无右栏版”的思路，重做聆听的前端主界面。

## v0.6 暂不在本阶段锁死的内容

虽然产品方向已经切换，但以下内容还不在当前文档里强行定稿：

- 新增哪些成熟产品能力
- 是否引入封面体系
- 是否引入更完整的艺术家、专辑维度
- 是否增加桌面端专属能力
- 是否重构现有后端 API 组织

这些可以在后续的 v0.6 详细设计文档里分别展开。

## 下一步

v0.6 的下一个动作应该是写页面重构设计，而不是继续修改 v0.5 文档。

建议以下顺序：

1. 固化“高级无右栏版”作为主界面方向。
2. 输出 v0.6 页面信息架构文档。
3. 明确歌曲页、媒体库页、歌单页、播放队列的职责边界。
4. 再进入真正的前端重构实施。
