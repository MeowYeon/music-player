# 阿言 v0.4 基础播放器功能设计

v0.4 的目标是补齐基础播放器体验，让阿言从“可扫描并播放本地歌曲”推进到“可以组织要听什么、按什么顺序听、保留基础听歌入口”。

## 功能目标

| 能力 | 说明 |
|---|---|
| 歌单管理 | 支持普通歌单创建、重命名、删除、添加歌曲、移除歌曲 |
| 我喜欢 | 作为系统歌单实现收藏能力 |
| 最近播放 | 作为系统歌单实现最近播放能力 |
| 播放队列 | 前端临时队列，右侧抽屉展示 |
| 播放顺序 | 支持顺序播放、列表循环、随机播放 |
| 统一歌曲行交互 | 全部歌曲、普通歌单、我喜欢、最近播放中的歌曲行行为一致 |

v0.4 不包含：

```text
歌词
封面
均衡器
导入导出歌单
云同步
跨端同步播放状态
持久化播放队列
```

## 数据库设计

新增 `playlists`：

| 字段 | 类型 | 说明 |
|---|---|---|
| id | INTEGER | 主键 |
| name | TEXT | 歌单名称 |
| type | TEXT | `normal`、`liked`、`recent` |
| created_at | TEXT | 创建时间 |
| updated_at | TEXT | 更新时间 |

新增 `playlist_music`：

| 字段 | 类型 | 说明 |
|---|---|---|
| playlist_id | INTEGER | 歌单 id |
| music_id | INTEGER | 音乐 id |
| added_at | TEXT | 加入歌单时间 |
| last_played_at | TEXT | 最近播放时间，仅最近播放系统歌单主要使用 |

约束：

| 约束 | 说明 |
|---|---|
| `UNIQUE(playlist_id, music_id)` | 同一歌单中同一首歌不重复出现 |
| `type = liked` 全局唯一 | 固定系统歌单 `我喜欢` |
| `type = recent` 全局唯一 | 固定系统歌单 `最近播放` |

## 歌单规则

| 歌单类型 | 行为 |
|---|---|
| 普通歌单 | 用户创建、重命名、删除；按加入时间展示歌曲 |
| 我喜欢 | 系统歌单；不可删除、不可重命名；收藏/取消收藏等价于加入/移出该歌单 |
| 最近播放 | 系统歌单；不可删除、不可重命名；播放开始后自动 upsert |

最近播放规则：

- 歌曲真正开始播放后写入最近播放。
- 重复播放同一首歌只更新 `last_played_at`，不重复插入。
- 展示时按 `last_played_at DESC`。
- 默认返回最近 50 首。
- v0.4 不做手动添加、清空最近播放和单条移除。

## API 设计

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/playlists` | 获取普通歌单列表，不包含系统歌单 |
| POST | `/api/playlists` | 创建普通歌单 |
| PATCH | `/api/playlists/{id}` | 重命名普通歌单 |
| DELETE | `/api/playlists/{id}` | 删除普通歌单 |
| GET | `/api/playlists/{id}/tracks` | 获取普通歌单内歌曲 |
| POST | `/api/playlists/{id}/tracks` | 添加歌曲到普通歌单 |
| DELETE | `/api/playlists/{id}/tracks/{trackId}` | 从普通歌单移除歌曲 |
| GET | `/api/playlists/liked/tracks` | 获取我喜欢歌曲 |
| POST | `/api/tracks/{id}/like` | 收藏歌曲 |
| DELETE | `/api/tracks/{id}/like` | 取消收藏歌曲 |
| GET | `/api/playlists/recent/tracks` | 获取最近播放 |
| POST | `/api/tracks/{id}/recent-play` | 记录最近播放 |

`GET /api/tracks` 返回歌曲时增加：

| 字段 | 类型 | 说明 |
|---|---|---|
| liked | boolean | 当前歌曲是否在 `我喜欢` 中 |

## 播放队列

播放队列为前端内存状态，刷新页面后丢失。

队列规则：

| 操作 | 行为 |
|---|---|
| 点击播放按钮 | 立即播放该歌曲，并插入播放队列队首 |
| 双击歌曲行 | 立即播放该歌曲，并插入播放队列队首 |
| 下一首播放 | 插入当前播放歌曲之后；如果当前没有播放歌曲，则加入队列末尾 |
| 移除队列歌曲 | 允许移除未播放歌曲 |
| 播放结束 | 根据当前播放顺序选择下一首 |

播放顺序：

| 模式 | 行为 |
|---|---|
| 顺序播放 | 播放到队列末尾后停止 |
| 列表循环 | 播放到队列末尾后回到第一首 |
| 随机播放 | 从队列中选择下一首，并避免立即重复当前歌曲 |

v0.4 不持久化播放队列、当前播放歌曲、播放进度、音量和播放模式。

## 统一歌曲行交互

以下列表中的歌曲行交互必须保持一致：

```text
歌曲管理
普通歌单详情
我喜欢
最近播放
```

统一行为：

| 交互 | 行为 |
|---|---|
| 点击播放按钮 | 播放该歌曲，并插入播放队列队首 |
| 单击歌曲行 | 只选中，不播放 |
| 双击歌曲行 | 播放该歌曲，并插入播放队列队首 |
| 收藏按钮 | 加入或移出 `我喜欢` |
| 三点菜单 | 提供 `添加到歌单`、`下一首播放` |

三点菜单只出现在歌曲行操作区。底部播放器不放更多菜单按钮。
