# 阿言 v0.3 数据库设计

v0.3 将媒体库、扫描状态、音乐详情、媒体库音乐关系拆开管理。扫描任务仍然独立成表，但它是媒体库的附属状态，不再作为多轮历史任务列表。

## 核心关系

| 关系 | 含义 |
|---|---|
| `libraries 1 -> 1 scan_tasks` | 每个媒体库只保留最近一次扫描状态 |
| `libraries N -> N music` | 一个媒体库包含多首音乐，一首音乐也可能被多个媒体库引用 |
| `library_music` | 管理媒体库与音乐的绑定关系 |

```text
libraries
  -> scan_tasks
  -> library_music
       -> music
```

## `libraries`

媒体库目录表，表示用户加入阿言管理的本地音乐目录。

| 字段 | 类型 | 约束 / 默认值 | 含义 |
|---|---|---|---|
| `id` | `INTEGER` | `PRIMARY KEY AUTOINCREMENT` | 媒体库 ID |
| `path` | `TEXT` | `NOT NULL UNIQUE` | 本地音乐目录绝对路径 |
| `created_at` | `TEXT` | 默认当前 UTC 时间 | 创建时间 |
| `updated_at` | `TEXT` | 默认当前 UTC 时间 | 更新时间 |

## `scan_tasks`

扫描任务表，与媒体库一对一，只保存该媒体库最近一次扫描信息。

| 字段 | 类型 | 约束 / 默认值 | 含义 |
|---|---|---|---|
| `id` | `INTEGER` | `PRIMARY KEY AUTOINCREMENT` | 扫描任务 ID |
| `library_id` | `INTEGER` | `NOT NULL UNIQUE`, FK -> `libraries.id` | 所属媒体库 |
| `status` | `TEXT` | `NOT NULL` | `idle`、`waiting`、`running`、`completed`、`failed` |
| `total_files` | `INTEGER` | `NOT NULL DEFAULT 0` | 最近一次扫描发现的音频文件数 |
| `scanned_files` | `INTEGER` | `NOT NULL DEFAULT 0` | 最近一次扫描已处理文件数 |
| `message` | `TEXT` | `NOT NULL DEFAULT ''` | 成功提示或失败信息 |
| `completed_at` | `TEXT` | 可为空 | 最近一次扫描结束时间 |

推荐提示文案：

| 场景 | `message` |
|---|---|
| 成功导入 | `扫描完成` |
| 源数据无变化 | `媒体库已是最新，无需重新导入` |
| 扫描失败 | 保存具体错误信息 |

## `music`

全局音乐详情表。展示全部歌曲列表时主要查询该表。

| 字段 | 类型 | 约束 / 默认值 | 含义 |
|---|---|---|---|
| `id` | `INTEGER` | `PRIMARY KEY AUTOINCREMENT` | 音乐 ID |
| `path` | `TEXT` | `NOT NULL UNIQUE` | 音频文件绝对路径，作为唯一判断 |
| `title` | `TEXT` | `NOT NULL` | 标题 |
| `artist` | `TEXT` | `NOT NULL DEFAULT ''` | 艺术家 |
| `album` | `TEXT` | `NOT NULL DEFAULT ''` | 专辑 |
| `duration_ms` | `INTEGER` | `NOT NULL DEFAULT 0` | 时长，毫秒 |
| `format` | `TEXT` | `NOT NULL` | 文件格式 |
| `size_bytes` | `INTEGER` | `NOT NULL` | 文件大小，byte |
| `mtime_unix` | `INTEGER` | `NOT NULL` | 文件最后修改时间，Unix 秒 |
| `created_at` | `TEXT` | 默认当前 UTC 时间 | 创建时间 |
| `updated_at` | `TEXT` | 默认当前 UTC 时间 | 更新时间 |

## `library_music`

媒体库与音乐的关系表。媒体库包含哪些音乐，只需要通过关系表表达，不在关系表中重复保存完整 metadata。

| 字段 | 类型 | 约束 / 默认值 | 含义 |
|---|---|---|---|
| `library_id` | `INTEGER` | `NOT NULL`, FK -> `libraries.id` | 媒体库 ID |
| `music_id` | `INTEGER` | `NOT NULL`, FK -> `music.id` | 音乐 ID |
| `created_at` | `TEXT` | 默认当前 UTC 时间 | 绑定创建时间 |

唯一约束：

```text
UNIQUE(library_id, music_id)
```

## 删除媒体库

删除媒体库时不删除本地音乐文件，只删除数据库索引。推荐在一个事务中执行：

```text
1. 查询该媒体库绑定的 music_id 列表
2. 删除该媒体库的 library_music 关系
3. 只检查第 1 步候选 music_id，清理已经没有任何媒体库引用的 orphan music
4. 删除 scan_tasks
5. 删除 libraries
```

不要先删除 `music`，因为音乐是否能删取决于移除当前媒体库关系后，它是否还被其他媒体库引用。

## 扫描与刷新

创建媒体库和扫描解耦：

```text
POST /api/libraries
  -> 只创建或复用媒体库

POST /api/libraries/{id}/scan
  -> 对指定媒体库触发扫描
```

前端创建媒体库成功后，可以主动触发第一次扫描，但后端创建媒体库接口本身不隐式扫描。

刷新策略：

| 请求 | 用途 | 建议频率 |
|---|---|---|
| `GET /api/libraries` | 低频同步媒体库列表和最近扫描摘要 | 约 5 秒 |
| `GET /api/tracks` | 低频同步音乐列表 | 约 15 秒，扫描完成后立即刷新 |
| `GET /api/scan-tasks/active` | 高频同步活跃扫描进度 | 约 2 秒，仅扫描中使用 |
