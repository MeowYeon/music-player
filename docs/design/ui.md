# 聆听 v0.6 界面设计基线

这份文档描述 v0.6 主界面重做后的当前界面事实。早期 MVP 界面不再作为当前基线。

## 总体布局

- 左侧为可折叠产品导航，展开态显示图标和文字，收起态为 icon-only。
- 顶部保留媒体库摘要和全局搜索。任意页面输入搜索会跳转到歌曲工作区结果状态。
- 主区域为无右栏工作区：歌曲、歌单、我喜欢、最近播放、媒体库。
- 底部是固定播放器。桌面显示完整控制；移动端默认显示迷你播放器，点击后展开完整控制。
- 播放队列由播放器入口打开：桌面为右侧抽屉，移动端为近全屏 sheet。

## 导航顺序

1. 歌曲
2. 歌单
3. 我喜欢
4. 最近播放
5. 媒体库

## 歌曲工作区

默认首页是成熟的歌曲工作区，不再使用“正在播放 + 数据统计”的控制台 hero。

- 顶部浏览启动区提供播放全部、随机播放、筛选摘要和搜索结果提示。
- 桌面保留表格优先，标题列聚合播放按钮、标题和轻量元信息。
- 行内操作保留播放、喜欢和更多菜单。
- 批量操作默认不常驻；选择歌曲后显示批量操作条。

## 歌单工作台

- 保留左侧歌单列表和右侧详情。
- 创建入口弱化为侧栏小表单。
- 右侧详情强化当前歌单标题、播放、重命名和删除。

## 我喜欢

- 作为收藏集合展示。
- 主动作偏播放收藏、随机播放和全部取消喜欢。
- 列表仍复用共享歌曲表格和批量操作。

## 最近播放

- 最近播放使用时间语义。
- API 返回 `{ track, lastPlayedAt }[]`。
- 页面按最新在上方展示，不做今天/昨天分组。
- 顶部展示轻量最近播放时间摘要。

## 媒体库

媒体库页从管理表格改为卡片列表。

每张卡片展示：

- 目录路径
- 歌曲数
- 扫描状态
- 扫描进度
- 错误或扫描消息
- 重新扫描

删除入口收进更多菜单，避免误触。
