# 阿言 v0.5 基础播放器闭环设计

v0.5 的目标是把 v0.4 已有的播放器能力补成更适合日常使用的闭环。范围保持轻量，不做歌词、封面、专辑页、艺术家页、实时监听、数据库迁移工具或桌面壳。

## 功能目标

| 能力 | 说明 |
|---|---|
| 播放全部 | 歌曲管理、普通歌单、我喜欢、最近播放都能一键播放当前列表 |
| 单曲循环 | 播放顺序增加 `single`，当前歌曲结束后重新播放当前歌曲 |
| 排序筛选 | 歌曲管理页支持按标题、艺术家、专辑、时长、格式排序，支持格式筛选和仅看我喜欢 |
| 多选批量操作 | 支持批量播放、下一首播放、添加到歌单、收藏、取消收藏 |
| 歌单移除 | 普通歌单详情页支持单首或批量移出歌曲 |
| 清空最近播放 | 最近播放页支持清空记录，不删除本地文件 |
| 播放状态记忆 | 使用浏览器 localStorage 记住音量、播放模式、队列和当前歌曲 |

## 播放队列规则

`播放全部` 会用当前页面实际展示的列表替换播放队列，并立即播放第一首。歌曲管理页会使用当前搜索、排序和筛选后的结果；歌单页会使用当前歌单歌曲；系统歌单页会使用当前系统列表。

播放模式：

| 模式 | 行为 |
|---|---|
| sequence | 播放到队列末尾后停止 |
| loop | 播放到队列末尾后回到第一首 |
| shuffle | 从队列中选择下一首，并避免立即重复当前歌曲 |
| single | 当前歌曲结束后重复播放当前歌曲 |

底部播放器按钮按 `顺序播放 -> 列表循环 -> 随机播放 -> 单曲循环 -> 顺序播放` 切换。

## 批量操作规则

多选只影响当前页面展示的歌曲列表。批量添加到歌单、收藏、取消收藏和移出歌单复用现有单项 API，由前端逐条调用。v0.5 不新增批量 API。

普通歌单详情页额外显示 `移出歌单` 操作；歌曲管理、我喜欢、最近播放不显示该操作。

## 最近播放清空

新增接口：

```text
DELETE /api/playlists/recent/tracks
```

返回 `204 No Content`。该接口只删除 `最近播放` 系统歌单中的 `playlist_music` 关系，不删除 `music`，不影响普通歌单和 `我喜欢`。

## 播放状态记忆

前端使用 localStorage key：

```text
ayan.player.v1
```

保存字段：

```text
volume
playMode
queueTrackIds
currentTrackId
```

刷新页面后恢复队列和当前歌曲，并保持暂停状态。v0.5 不记忆播放进度，也不在刷新后自动播放，避免浏览器自动播放限制和状态歧义。
