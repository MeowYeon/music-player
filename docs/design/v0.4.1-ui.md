# 阿言 v0.4.1 UI 对齐设计

v0.4.1 目标是将现有 Web 界面向参考图的视觉规范对齐。本次只调整前端 UI 和交互细节，不改后端数据模型。

## 设计方向

| 项目 | 要求 |
|---|---|
| 整体 | 白底、细线、低阴影、紧凑播放器工具感 |
| 字体 | 系统字体，减少粗体，提升信息密度 |
| 主色 | 深薄荷绿 `#14966f` |
| 边框 | 浅灰细线 `#e6e8eb` |
| 圆角 | 8px 到 12px |
| 阴影 | 仅菜单、弹窗、队列浮层使用轻阴影 |

## 侧边栏

- 宽度调整到更接近参考图比例，展开态约 `260px`。
- 品牌区展示 `阿言` 和 `v0.4` badge。
- 收起按钮保留在品牌区右侧。
- 导航项图标和文本横向排列。
- 导航项高度约 `56px`，左右 padding 约 `16px`。
- 选中态使用浅绿背景，文字和图标使用主绿。
- 侧边栏只保留右侧细分割线，不使用明显阴影。

## 顶部区域

- 左侧展示页面标题，例如 `歌曲管理`。
- 标题下方展示摘要，例如 `3 个媒体库 · 128 首歌曲`。
- 搜索框放右上，宽度约 `360px - 400px`。
- 搜索框高度约 `44px`，浅灰边框，左侧搜索图标。
- 顶部区域不使用卡片容器。

## 歌曲表格

表格采用细线分割和紧凑行高：

| 元素 | 要求 |
|---|---|
| 表头 | 高度约 46px，浅灰文字 |
| 表格行 | 高度约 56px 到 58px |
| 播放按钮 | 32px 圆形描边按钮 |
| 当前播放行 | 浅绿背景，关键文字使用主绿 |
| 长文本 | 标题、路径必须单行省略 |
| 三点菜单 | 只保留两个选项：`添加到歌单`、`下一首播放` |

推荐列：

```text
勾选
播放
标题
歌手
专辑
时长
格式
路径
喜欢
更多
```

## 添加到歌单弹窗

- 弹窗宽度约 `430px - 460px`。
- 标题为 `添加到歌单`。
- 副标题显示当前歌曲名，长标题省略。
- 提供搜索歌单输入框。
- 已有歌单使用单选列表展示。
- 下方提供 `新建歌单并添加`：
  - 输入歌单名称。
  - 点击 `创建并添加` 后创建歌单并添加当前歌曲。
- 底部按钮为 `取消` 和 `添加到选中歌单`。
- 三点菜单中不再展开所有歌单。

## 底部播放器

- 底部播放器为全宽固定底栏。
- 高度约 `86px - 96px`。
- 左侧增加封面图区域，尺寸约 `64px x 64px`。
- 当前歌曲标题固定宽度，长标题省略。
- 收藏心形靠近当前歌曲信息。
- 播放控制按钮靠左中区域。
- 进度条细线化，使用主绿。
- 播放顺序按钮放在进度条右侧。
- 音量和播放队列按钮处在同一水平线。
- 播放队列按钮位于最右侧，并使用描边绿按钮。

## 播放队列浮层

- 播放队列不使用侧边抽屉。
- 从右下角播放队列按钮上方弹出。
- 宽度约 `320px - 360px`。
- 带小三角指向队列按钮。
- 不遮挡底部播放器。
- 当前播放行使用浅绿背景。
- 底部展示 `共 N 首` 和 `清空队列`。

## 不做项

```text
不改数据库模型
不新增后端 API
不做封面读取
不做主题切换
不做复杂响应式重排
```
