# 聆听 v0.6 主界面重做实施计划

Status: planned
Baseline: v0.5
Created: 2026-05-10

## Summary

先把当前未提交状态整理为 **v0.5 基线提交并打 tag `v0.5`**，然后按小版本推进 v0.6 主界面重做。每个功能点独立 commit；每个小版本合并回 `main` 后打 tag。允许并行多 agent，但必须通过独立 branch + worktree 降低冲突。

## Version Plan

### v0.5：当前状态基线

- 目标：把当前工作区整理为 v0.5 收尾提交。
- 操作：
  - 检查当前未提交文件，确认它们属于 v0.5 已完成成果。
  - 创建一个基线提交，例如：`chore: mark v0.5 baseline`
  - 打 tag：`v0.5`
- 约束：
  - v0.6 工作不得直接建立在脏工作区上。
  - v0.5 提交完成后，`main` 必须干净。

### v0.6.1：应用骨架、导航、搜索、CSS 基线

- 目标：建立 v0.6 主界面骨架，不深入重做所有页面内容。
- 功能 commit：
  1. `refactor: clean legacy app styles`
     - 清理过期 prototype CSS。
     - 收敛重复 `.track-table` 样式策略。
  2. `feat: add collapsible product navigation`
     - 侧栏支持展开/收起。
     - 展开态图标 + 文字，收起态 icon-only。
     - 导航顺序改为：歌曲 / 歌单 / 我喜欢 / 最近播放 / 媒体库。
  3. `feat: make search global`
     - 顶部搜索成为全库搜索。
     - 任意页面输入搜索后跳转到歌曲工作区结果状态。
  4. `refactor: split app shell components`
     - 适度拆出应用壳、导航、顶部栏等组件。
- tag：`v0.6.1`

### v0.6.2：歌曲工作区

- 目标：把默认首页重做为成熟的歌曲工作区。
- 功能 commit：
  1. `feat: redesign songs workspace hero`
     - 去掉“正在播放 + 数据统计”控制台 hero。
     - 改为浏览启动区：播放全部、随机播放、筛选摘要、搜索结果提示。
  2. `refactor: refine desktop track table`
     - 保留桌面表格优先。
     - 优化列宽、行内播放/喜欢/更多操作。
     - 降低后台数据表视觉。
  3. `feat: show batch actions after selection`
     - 默认不常驻批量按钮。
     - 选中歌曲后显示批量操作条。
  4. `refactor: extract shared track list components`
     - 拆出歌曲表格、批量操作条、空状态/错误状态。
- tag：`v0.6.2`

### v0.6.3：歌单工作台、我喜欢、最近播放 API

- 目标：重做歌单与系统歌单页面，并让最近播放具备时间语义。
- 功能 commit：
  1. `feat: add recent track item response`
     - `GET /api/playlists/recent/tracks` 返回 `{ track, lastPlayedAt }[]`。
     - 普通歌单和我喜欢继续返回 `Track[]`。
  2. `feat: redesign recent plays page`
     - 最近播放按最新在上方展示。
     - 不做今天/昨天分组。
     - 可展示轻量播放时间。
  3. `feat: redesign liked tracks page`
     - 强化收藏集合语义。
     - 主动作偏播放/随机播放/取消喜欢。
  4. `feat: redesign playlist workbench`
     - 保留左侧歌单列表 + 右侧详情。
     - 弱化创建入口。
     - 强化当前歌单标题、播放、重命名、删除。
- tag：`v0.6.3`

### v0.6.4：媒体库卡片与播放器/队列响应式

- 目标：完成低频管理页和播放器体验整理。
- 功能 commit：
  1. `feat: replace library table with cards`
     - 媒体库页改为库卡片列表。
     - 卡片展示路径、歌曲数、扫描状态、进度、错误、重新扫描。
     - 删除入口放入更多菜单。
  2. `feat: add mobile mini player`
     - 桌面保留底部固定完整播放器。
     - 移动端显示迷你播放器。
     - 点击后展开完整控制。
  3. `refactor: adapt queue drawer responsively`
     - 桌面仍为右侧抽屉。
     - 移动端表现为近全屏 sheet。
  4. `test: cover v0.6 interface flows`
     - 补充关键 UI 状态、最近播放 API、搜索跳转、批量操作测试。
- tag：`v0.6.4`

### v0.6.5：文档、回归、发布整理

- 目标：收尾 v0.6 主界面重做。
- 功能 commit：
  1. `docs: document v0.6 interface architecture`
     - 更新设计文档、架构文档、工程导览。
  2. `test: run v0.6 regression suite`
     - 前端 build/typecheck/test。
     - 后端 Go test。
  3. `chore: remove obsolete v0.6 leftovers`
     - 移除未使用样式、旧组件、废弃注释。
- tag：`v0.6.5`

## Branch / Worktree / Agent Strategy

- 主分支：
  - `main` 只接收已验收的小版本合并。
  - 每个小版本完成后合并回 `main` 并打 tag。
- 小版本分支：
  - `codex/v0.6.1-shell`
  - `codex/v0.6.2-songs`
  - `codex/v0.6.3-playlists`
  - `codex/v0.6.4-library-player`
  - `codex/v0.6.5-release`
- worktree 目录：
  - 使用 `/tmp/music-worktrees/v0.6.1-shell`
  - 使用 `/tmp/music-worktrees/v0.6.2-songs`
  - 使用 `/tmp/music-worktrees/v0.6.3-playlists`
  - 使用 `/tmp/music-worktrees/v0.6.4-library-player`
- 并行原则：
  - v0.6.1 必须先完成，因为它清理 CSS 和应用骨架。
  - v0.6.2、v0.6.3、v0.6.4 可在 v0.6.1 后并行，但必须拆不同 worktree。
  - 同一文件高冲突区域需要指定 owner：
    - Shell/navigation/search owner：应用壳和全局状态。
    - Songs owner：歌曲工作区和共享歌曲列表。
    - Playlists owner：歌单、我喜欢、最近播放、最近播放 API。
    - Library/player owner：媒体库卡片、播放器、队列。
  - 多 agent 不允许互相回滚对方改动；集成时只解决自己的冲突范围。

## Commit / Tag Rules

- 每个功能点一个独立 commit。
- commit message 使用：
  - `feat: ...`
  - `refactor: ...`
  - `test: ...`
  - `docs: ...`
  - `chore: ...`
- 每个小版本合并回 `main` 后打 tag：
  - `v0.5`
  - `v0.6.1`
  - `v0.6.2`
  - `v0.6.3`
  - `v0.6.4`
  - `v0.6.5`
- tag 前必须满足：
  - 工作区干净。
  - 当前小版本测试通过。
  - 小版本内 commit 粒度符合功能拆分。
  - 文档与实际行为不冲突。

## Test Plan

- 每个小版本至少执行：
  - 前端类型检查/build。
  - 相关前端测试。
  - 后端 Go test，如该版本改了 API/storage。
- v0.6.3 必须额外验证：
  - 最近播放接口返回 `{ track, lastPlayedAt }[]`。
  - 最近播放按最新在上方。
  - 普通歌单和我喜欢仍按原 `Track[]` 行为工作。
- v0.6.4 必须额外验证：
  - 媒体库空/扫描中/完成/失败状态。
  - 桌面播放器、移动迷你播放器、队列抽屉。
- v0.6.5 做完整回归。

## Assumptions

- 当前未提交改动将先整理为 v0.5 基线提交。
- v0.6 不新增发现首页、不引入封面系统。
- v0.6 桌面优先，移动端保证可用。
- 最近播放是唯一需要时间语义的列表。
- 多 agent 只在 v0.6.1 之后并行，避免 CSS/应用壳基线冲突。
