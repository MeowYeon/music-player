# 聆听项目文档

这里是聆听项目的文档入口。文档按用途分层，避免把“当前事实”“协作规则”“设计方向”和“历史记录”混在一起。

## 快速阅读顺序

新协作者或模型接手项目时，建议按这个顺序阅读：

1. [项目记忆说明](./memory.md)
2. [工程导览](./engineering.md)
3. [当前架构](./design/architecture.md)
4. [设计文档索引](./design/README.md)

## 文档分层

### 协作记忆

- [项目记忆说明](./memory.md)
  - 解释项目文档如何组成长期上下文。
  - 说明 agent / GPT 的推荐阅读顺序。

- [根目录 AGENTS.md](../AGENTS.md)
  - 面向自动化协作者的第一入口。
  - 记录当前环境、项目入口和服务命令。

### 当前事实

- [工程导览](./engineering.md)
  - 说明代码目录、模块入口、数据流和常用命令。

- [当前架构](./design/architecture.md)
  - 描述当前实现的系统架构、数据模型和 API。

- [当前界面设计](./design/ui.md)
  - 描述当前界面设计基线。v0.6 重构后需要更新。

### 产品与设计

- [设计文档索引](./design/README.md)
  - 版本设计、阶段取舍和图片资料入口。

- [v0.6 产品方向](./design/v0.6-product-direction.md)
  - 说明聆听从 MVP 走向成熟产品的方向。

### 历史记录

- [版本历史](./history.md)
  - 记录早期版本完成情况和历史验收。
  - 不作为当前计划入口。

## 维护原则

- `AGENTS.md` 保持简短，偏向运行环境和项目入口。
- `docs/memory.md` 解释文档系统本身，帮助快速建立项目上下文。
- `docs/engineering.md` 只写当前代码事实，不写产品愿景。
- `docs/design/` 放产品、架构、界面和版本设计。
- `docs/history.md` 保留历史记录。
